💿 내장함수 중 map()

파이썬의 내장함수를 배웠는데 그 중 map() 함수가 기억나서 적어본다.

map(f, iterable)의 형태를 가지고 있기에, 함수를 넣고 옆에는 반복 가능 자료형을 넣으면 알아서 함수를 수행하고 그 결과값을 한 번에 주는 줄 알았다.

뭐 틀린 말은 아니긴 한데 말이지....

```python
def mul(a):
  result = []
  for i in a:
    result.append(i*2)
  return result

list(map(mul, [1, 2, 3, 4]))  #map() 함수는 filter() 함수처럼 그 결과를 객체에 반환하기만 해서 우리가 보려면 list()처럼 처리를 해줘야한다고 함
```

이렇게 한 번에 해봣는데, 오류가 나는거다, 하....

이거 뭐지 뭐지 하고 계속 보는데 도저히 모르겠더라, 아니 함수도 잘 썼고, map() 함수 안에 함수와 반복 가능한 자료형도 잘 집어넣었는데 뭐가 문제인지 통 감이 안잡혔다..

근데 보니까 map()은 입력 받은 자료형의 '각 요소'를 함수가 수행한 결과를 묶어서 돌려준다는 설명이 눈에 들어왔다.

이게 그러니까 내가 map 함수에 넣은 [1, 2, 3, 4]라는 리스트를 통째로 가져가서 mul 함수에 집어 넣는게 아니라 그 리스트에 빼서 하나씩 넣는거였다

근데 또 궁금한게 그러면 "mul([1]) 아님??" 이라는 의문이 들었다.

근데 생각해보면 [1, 2, 3, 4]는 리스트 하나고, [1], [2], [3], [4]는 리스트 네개이고, 그리고 우리가 [1, 2, 3, 4]에서 인덱싱 0번을 하면 1이 나오지 [1]이 나오지 않잖아?

그래서 아 요소는 리스트 안에 들어있는 값의 순수한(?) 그 원래의 값과 형태구나라고 이해했다.

즉, 내가 위의 mul 함수에 넣은건 리스트 [1]이 아니라 일반 정수 1이었던 것이다. 그러니까 반복 가능한 자료형이 아니니까 for문이 작동하지 않은거였다...

그러니까 정수형을 함수에 대입했을 때 잘 작동하려면

```python
def mul(a):
  return a*2

list(map(mul, [1, 2, 3, 4]))  #결과값은 [2, 4, 6, 8] 도출
```

이렇게 mul 함수에 정수가 들어가도 가능하게 만드는 것이었다.

근데 뭔가 좀 기존 함수에서도 잘 작동할 수 있게 하는 방법이 없을까라는 생각이 들었다. 그래서 생각난게

```python
def mul(a):
  result = []
  for i in list(a):
    result.append(i*2)
  return result
```

이거였는데 list 함수는 literable한 것만 받고

```python
def mul(a):
  result = []
  for i in [a]:
    result.append(i*2)
  return result
```

이거는 [[2], [4], [6], [8]]이 나와 기존 값이랑 달랐다.

결국 map함수를 이용해서 같은 값을 하는건 실패했는데 애당초 [1], [2], [3], [4]를 주면 되겠지만 그러면 [1, 2, 3, 4]랑 다르니까...

그래서 굳이 억지로 안끼어넣고 그냥 a = mul([1, 2, 3, 4])로 하면 될 것 같다. map을 쓸꺼면 내가 그거에 맞춰주고...
