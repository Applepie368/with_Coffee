💿 '예외 처리'

일부러 오류를 무시하고, 혹은 명령어 raise까지 동원해서 일부러 오류를 내는 등에서 사용하는 방식

오류를 일부러 회피할 때는 게임 새로 시작시, 기존의 파일을 자동 불러올 때 불러올 수 없으니(파일이 없으니) 무시하고 넘어가는 등에서 쓴다고 한다

뭐 근데 

```python
try:
  내용
except:
  pass
```

요거는 별로 좋지 않다고 한다. 왜냐면 무지성 pass라서, 특정 오류를 골라서 넘어가는게 아니라 오류가 나면 다 무시하고, 그리고 무슨 오류인지도 알려주지 않아서 좋지 않다고 한다

정확히 특정 오류에 대해서 무시하거나, 혹은 무시하지만, 어떤 오류가 났는지 알려주는 것이 도움이 되고 자주 쓴다고 한다

```python
try:
  내용
except IndexError:  
  pass
```

```python
try:
  내용
except IndexError as E:
  print(E)
```

처럼 말이다

내가 가장 흥미 있었고, 좀 헷갈렸던 것은 명령어 raise로 내가 직접 오류를 만들어서 예외를 만들 때였다

일단은 내가 파이썬 기존의 클래스인(오류와 관련된) Exception을 상속함으로써 시작이 되었다

```python
class MyError(Exception): #MyError는 내가 오류로 설정할 클래스
  pass
```

그리고 내가 Sora에게 별명을 지어줄껀데, 바보라고 하면 MyError라는 오류가 발생하게 하려고 한다

```python
def Sora(name):
  if name == "바보":
    raise MyError()
  print(name)
```

이걸 예외처리 기법으로 실행해보면

```python
try:
  name("천사")
  name("바보")
except MyError:
  print("불허함")
```

이러면 결과값으로 천사와 불허함이라는 말이 출력된다 그렇다면 특정 객체 e에 넣고 출력하는 방식은?

```python
except MyError as e:
  print(e)
```

이러면 천사라는 낱말과 아무것도 없는 공백이 출력된다.


여기서 처음으로 __str__이라는 걸 접하게 되었다. 이전에 str이 문자열과 관련된걸 java에서도 python에서도 눈에 익혔기 때문에 뭔지는 알았다 근데

```python
class MyError(Exception):
  def __str__(self):
    return "불허함"
```

이라고 해놓아야 print(e)에서 우리가 의도한 대로 출력이 된다고 하는 것에서 이해가 안되었다.
return 뒤에 바로 따옴표와 문자열이 있는것도 어색하고, __str__로 함수를 놓는 것 대신 그냥 print("~")를 쓰면 안되나 하는 의문이 생겼다

claude랑 계속 얘기하면서 알아본 결과

__str__은 해당 객체를 어떻게 문자열로 정의할 것인지를 묻는 것이었고, 그래서 결과값 반환시 문자열이 필요하다고 했다

근데 그럴거면 귀찮게 그러지 말고 print() 쓰는건 어떤지 하고 테스트 해봤는데, 오류가 뜨기도 전에 print()의 안에 있는 내용이 출력되었다.

그래서 보니 return으로 가지고 있다가 필요할 때 꺼내야지, print()쓰면 내가 직접 실행시키지 않아도 파이썬이 읽어내려가면서 바로 실행시켜버린다고 해서, 해당 클래스가 상속할 때 바로 써넣으면 안된다는걸 알았다


그리고 또 한가지 방법이 있는데,

상속 받을 때는 그대로 두고, 함수 때에 raise 명령어로 설정한 클래스를 호출시 그 안에 내가 출력할 메시지를 넣는 것이다

```python
def Sora(name):
  if name == "바보":
    raise MyError("불허함")
  print(name)
```

또... 이게 뭔소리야.... MyError가 Exception을 상속받은거 ok, 근데 뭔 문자열 인수를 넣어서 출력이 되는게 어떻게 되는거야....ㅠㅠㅠ

알고 봤더니, 애당초 파이썬 내장 클래스인 Exception에서 __init__에서 (self, message = "")라는 매개변수가 설정되었다고 한다

평소에는 기본값이 설정되었지만, 내가 새로운 값을 주었으니 그걸 받아서 가지고 있다가, 그걸 e라는 객체에 전달하라고 할 때 전달해주고, print(e)를 함으로써 나타나는거지

그러니까...

```python
class MyError(Exception):
  pass

def Sora(name):
  if name == "바보":
    raise MyError("불허함")
  print(name)

try:
  name("천사")
  name("바보")
except MyError as e:
  print(e)
```

라고 했을 때 결과값은

```python
천사
불허함
```

이렇게 나오게 된다는거지


그러면 또 궁금한게 애당초 클래스 상속시 에러나 났을 때 출력할 값을 정할지, 함수에서 정하는 방법 중 뭐가 실전에서 많이 쓰일까 궁금해서 물어봤다

답은 함수(메서드)에서 설정하는게 좀 더 많이 쓰인다고 한다

왜냐면 클래스 상속에서 설정시 내가 설정한 오류 클래스를 적용한 곳에서 모두 통일된 값이 나오지만, 함수 마다 적을경우 내가 특정한 값에서는 다른 내용을 적을 수 있으니까 말이다
